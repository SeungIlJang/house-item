# 실행 방법 (집안의모든것 / home-items)

이 문서 하나로 로컬 개발 실행 → 접속 → 검사/테스트 → Docker 실행까지 따라 할 수 있습니다.

- 작업 폴더: `/Users/seungiljang/workspace/house-items`
- 백엔드: `home-items-api` (FastAPI, 포트 8000)
- 프론트: `home-items-app` (Ionic Vue, 포트 5173)
- DB: 로컬 PostgreSQL (포트 5432)

> 계정/비밀번호 등은 `CREDENTIALS.local.md`(git 제외) 참고.

## 0. 사전 준비 (최초 1회)

### 0-1. 필요한 도구
| 도구 | 확인 명령 | 비고 |
|------|-----------|------|
| Python 3.13+ | `python3 --version` | |
| uv | `uv --version` | 파이썬 패키지 관리 |
| Node 18+ (20 권장) | `node --version` | |
| PostgreSQL 14+ | `psql --version` | 로컬 실행 중이어야 함 |

### 0-2. 데이터베이스 만들기 (최초 1회)
```bash
psql -d postgres -c "CREATE ROLE home_items LOGIN PASSWORD 'home_items_dev_pw';"
psql -d postgres -c "CREATE DATABASE home_items OWNER home_items;"
```

### 0-3. 환경 변수 파일 (최초 1회)
```bash
cd /Users/seungiljang/workspace/house-items
cp .env.example .env
# (권장) JWT 비밀키를 무작위 값으로 교체

#   → 출력값을 .env 의 JWT_SECRET_KEY 에 붙여넣기
```

## 1. 백엔드 실행

```bash
cd /Users/seungiljang/workspace/house-items/home-items-api

uv sync                                # 의존성 설치 (최초 1회/변경 시)
uv run alembic upgrade head            # DB 테이블 생성/최신화 (최초 1회/모델 변경 시)
uv run uvicorn app.main:app --reload   # 개발 서버 시작
```

- 서버: http://localhost:8000
- API 문서(Swagger): http://localhost:8000/docs
- 상태 확인: http://localhost:8000/health , http://localhost:8000/health/db

중지: 터미널에서 `Ctrl + C`

---

## 2. 프론트 실행

새 터미널을 열고:

```bash
cd /Users/seungiljang/workspace/house-items/home-items-app

npm install       # 의존성 설치 (최초 1회/변경 시)
npm run dev       # 개발 서버 시작
```

- 앱: http://localhost:5173
- 브라우저에서 회원가입 → 로그인 → 집/방/수납공간 등록 → 물건 등록/검색

중지: 터미널에서 `Ctrl + C`

> 백엔드(8000)와 프론트(5173)를 **둘 다** 실행해야 정상 동작합니다.

## 3. 첫 사용 순서 (앱에서)

1. 회원가입 후 로그인
2. 하단 탭 **보관 장소** → 집 등록 → 방 등록 → 수납공간 등록
3. (선택) **내 정보** → 카테고리/태그 등록
4. 하단 탭 **등록** → 물건 정보 입력 + 사진 → 저장
5. 하단 탭 **찾기** → 이름/카테고리로 검색

## 5. Docker 로 전체 실행 (배포/통합, 선택)

> 개발용은 위 1·2번을 쓰세요. Docker 는 배포 또는 한 번에 전체 실행이 필요할 때만 사용합니다.
> (Docker Desktop 설치 필요)

```bash
cd /Users/seungiljang/workspace/house-items
docker compose up --build
```
- 프론트: http://localhost:8080
- 백엔드: http://localhost:8000/docs

중지: `Ctrl + C` 후 `docker compose down` (데이터 볼륨까지 지우려면 `docker compose down -v`)

## 6. 자주 겪는 문제 (Troubleshooting)

| 증상 | 원인 / 해결 |
|------|-------------|
| 백엔드 `uv run` 이 `alembic not found` | 작업 폴더가 `home-items-api` 인지 확인 후 실행 |
| `/health/db` 500 | PostgreSQL 미실행 또는 `.env`의 `DATABASE_URL` 확인 |
| 프론트에서 로그인/요청 실패 | 백엔드(8000)가 실행 중인지, `.env`의 `VITE_API_BASE_URL` 확인 |
| CORS 오류 | 백엔드 `.env`의 `CORS_ORIGINS` 에 프론트 주소(`http://localhost:5173`) 포함 |
| 포트 충돌(5432/8000/5173) | 사용 중인 프로세스 종료 또는 포트 변경 |
| DB 초기화하고 싶음 | `psql -d postgres -c "DROP DATABASE home_items;"` 후 재생성 + `alembic upgrade head` |
