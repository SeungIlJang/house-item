## 2. 설계 설명

| 구성 | 선택 / 이유 |
|------|-------------|
| 비밀번호 해시 | **bcrypt 직접 사용** (`passlib`가 bcrypt 5.0과 호환 안 됨 → 전환). 평문 저장 금지 |
| 비밀번호 길이 | 스키마에서 8~72자 제한(bcrypt 72바이트 한계) |
| 토큰 | JWT Access Token (`sub`=user id, 만료 60분). Refresh Token은 MVP 이후 |
| 로그인 실패 | 이메일 존재 여부를 노출하지 않도록 동일 메시지("이메일 또는 비밀번호가 올바르지 않습니다") |
| 인증 헤더 | `Authorization: Bearer <token>` (HTTPBearer, Swagger UI 지원) |
| 계층 | Router → Service(AuthService) → Repository(UserRepository) → DB |
| 응답 | 비밀번호/해시는 응답 스키마(UserResponse)에 절대 포함하지 않음 |

> 용어 한 줄 설명
> - **bcrypt**: 비밀번호 전용 단방향 해시 함수. 같은 비밀번호도 매번 다른 해시(salt).
> - **JWT sub**: 토큰 주체(subject). 여기서는 사용자 id.

## 4. 실행 명령어

```bash
cd home-items-api
uv sync
uv run alembic revision --autogenerate -m "create users table"
uv run alembic upgrade head
uv run ruff check .
uv run pytest -q
```

### API 사용 예 (Swagger: http://localhost:8000/docs)
```bash
# 회원가입
curl -X POST http://localhost:8000/api/v1/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123","name":"홍길동"}'

# 로그인 → accessToken 획득
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}'

# 내 정보
curl http://localhost:8000/api/v1/users/me -H "Authorization: Bearer <accessToken>"
```

## 5. 실행과 테스트 결과

- 마이그레이션: `Detected added table 'users'`, index `ix_users_email` → upgrade 성공 ✅
- `ruff check` : **All checks passed!**
- `pytest` : **9 passed** (health 2 + auth 7)
  - 회원가입 성공(비밀번호 응답 미포함 확인)
  - 이메일 중복 → 409 `EMAIL_ALREADY_EXISTS`
  - 잘못된 이메일 → 422 `VALIDATION_ERROR`
  - 로그인 성공 → 토큰 발급 → `/users/me` 조회 성공
  - 잘못된 비밀번호 → 401 `INVALID_CREDENTIALS`
  - 토큰 없음 → 401 `NOT_AUTHENTICATED`
  - 잘못된 토큰 → 401 `INVALID_TOKEN`

### 이번 단계에서 해결한 문제
1. **passlib × bcrypt 5.0 비호환** (`module 'bcrypt' has no attribute '__about__'`) → bcrypt 직접 사용으로 전환.
2. **EmailStr 의존성 누락** → `pydantic[email]`(email-validator) 추가.
3. **상위 uv 워크스페이스 충돌** (`Workspace member house-items missing pyproject`)
   → 루트에 최소 `pyproject.toml` 복원 + 백엔드를 자체 워크스페이스로 격리.
4. **테스트에서 `app` 이름 충돌** (`import app.models`가 FastAPI 인스턴스 이름을 덮음)
   → 인스턴스를 `fastapi_app`으로 import.
