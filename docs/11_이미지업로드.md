# 11단계: 이미지 업로드

> 집안의모든것(home-items) 프로젝트 · 작성일 2026-07-17

---

## 1. 이번 단계의 목표

- 물건 이미지 업로드 / 조회 / 삭제
- 대표 이미지(썸네일)
- 파일 검증(확장자·MIME·크기)
- 저장 인터페이스 분리(Local/S3 교체 대비)

---

## 2. 설계 설명

| 구성 | 내용 |
|------|------|
| 저장 추상화 | `FileStorage`(추상) → `LocalFileStorage`(개발), `S3FileStorage`(배포 예정 stub). DB에 BLOB 저장 안 함 |
| 저장 경로 | `uploads/items/{user_id}/{item_id}/{uuid}.ext`, URL `/uploads/...` |
| 정적 제공 | FastAPI `StaticFiles`로 `/uploads` 마운트 |
| 검증 | 확장자(jpg/jpeg/png/webp/gif), MIME(image/*), 업로드 원본 최대 10MB, 빈 파일 금지 |
| 리사이즈/압축 | 저장 전 긴 변 최대 1600px 축소 + 재인코딩(JPEG 품질 85). 투명 이미지는 PNG 유지, GIF는 원본 유지. EXIF 방향 보정. 예) 약 3MB → 약 400KB(≈86% 절감) |
| 대표 이미지 | `sort_order` 최소(먼저 업로드한 것) = 썸네일. 물건 응답에 `thumbnailUrl` 제공 |
| 소유권 | 업로드/삭제 시 물건 소유권 확인 |
| 이름 주의 | 파일 저장소(FileStorage)와 수납공간(StorageLocation)은 다른 개념 |

## 4. API 목록

```text
POST   /api/v1/items/{item_id}/images            (multipart: file)

```
