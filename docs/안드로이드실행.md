# 안드로이드 폰에서 실행하기 (Capacitor)

집안의모든것 웹앱을 Capacitor 로 감싸 안드로이드 폰에서 실행합니다.

- 이미 빌드된 APK: `~/Desktop/집안의모든것.apk` (또는
  `home-items-app/android/app/build/outputs/apk/debug/app-debug.apk`)
- 이 APK 는 **개발 PC 의 백엔드(http://172.30.1.46:8000)** 를 바라보도록 빌드되었습니다.

> ⚠️ **중요**: 앱은 폰에서 실행되지만 데이터는 **개발 PC 의 백엔드**에서 옵니다.
> 따라서 **PC 백엔드가 켜져 있어야** 하고, **폰과 PC 가 같은 Wi‑Fi** 에 있어야 합니다.

---

## 1. 백엔드를 폰에서 접속 가능하게 실행

PC 에서 백엔드를 **0.0.0.0** 으로 실행합니다(외부 접속 허용):

```bash
cd home-items-api
uv run uvicorn app.main:app --host 0.0.0.0 --port 8000
```

- 확인: PC 브라우저에서 `http://172.30.1.46:8000/health` 가 열리면 OK
- macOS 방화벽이 차단하면 "허용"을 눌러주세요(시스템 설정 → 네트워크 → 방화벽).

> PC 의 IP(172.30.1.46)는 Wi‑Fi 가 바뀌면 달라질 수 있습니다.
> 바뀌면 아래 "IP 가 바뀌었을 때"를 참고해 다시 빌드하세요.

---

## 2-A. 방법 A — USB 로 폰 연결해서 설치 (권장)

1. 폰에서 **개발자 옵션 → USB 디버깅** 켜기
   - (설정 → 휴대전화 정보 → 소프트웨어 정보 → **빌드번호 7번 탭** → 개발자 옵션 활성화)
2. 폰을 USB 로 PC 에 연결 → 폰에 뜨는 "USB 디버깅 허용" 팝업 **허용**
3. PC 에서:
   ```bash
   ~/Library/Android/sdk/platform-tools/adb devices     # 기기가 보이는지 확인
   ~/Library/Android/sdk/platform-tools/adb install -r ~/Desktop/집안의모든것.apk
   ```
4. 폰 앱 서랍에서 **집안의모든것** 실행

## 2-B. 방법 B — APK 파일을 폰으로 옮겨 설치

1. `~/Desktop/집안의모든것.apk` 를 카카오톡/이메일/USB/구글드라이브 등으로 폰에 전송
2. 폰에서 그 파일을 탭 → "출처를 알 수 없는 앱 설치 허용" 요청 시 허용 → 설치
3. **집안의모든것** 실행

---

## 3. 사용

1. 앱에서 회원가입 → 로그인
2. 가입 시 기본 집/보관장소/수납공간/카테고리/태그가 자동 생성됨
3. 물건 등록/검색/사진 업로드 확인

> 로그인/요청이 안 되면: (1) PC 백엔드가 켜져 있는지, (2) 폰·PC 가 같은 Wi‑Fi 인지,
> (3) `http://172.30.1.46:8000/health` 가 PC 에서 열리는지 확인하세요.

## IP 가 바뀌었을 때 (또는 다른 네트워크)

PC 의 IP 확인:
```bash
ipconfig getifaddr en0
```

새 IP 로 다시 빌드/설치:
```bash
cd home-items-app
VITE_API_BASE_URL=http://<새IP>:8000/api/v1 npm run build
npx cap sync android
cd android && JAVA_HOME=$(/usr/libexec/java_home -v 17) ./gradlew assembleDebug
cp app/build/outputs/apk/debug/app-debug.apk ~/Desktop/집안의모든것.apk
# 다시 방법 A/B 로 설치
```

## 처음부터 다시 만들 때 (참고)

`android/` 폴더는 git 에 포함하지 않습니다(재생성 가능). 새로 만들 때:

```bash
cd home-items-app
npm install                                   # capacitor 포함 의존성
VITE_API_BASE_URL=http://<IP>:8000/api/v1 npm run build
npx cap add android                           # android 프로젝트 생성
```

그리고 `android/app/src/main/AndroidManifest.xml` 를 수정합니다.

1) **HTTP(cleartext) 허용** — `<application>` 태그에 속성 추가:
```xml
android:usesCleartextTraffic="true"
```
(개발 PC 의 http 백엔드 접속용. `capacitor.config.ts` 의 `server.cleartext: true` 와 함께)

2) **카메라 권한** — `<manifest>` 바로 아래에 추가:
```xml
<uses-permission android:name="android.permission.CAMERA" />
```
(사진 촬영 기능용. `@capacitor/camera` 플러그인 필요: `npm install @capacitor/camera`)

이후:
```bash
npx cap sync android
cd android && JAVA_HOME=$(/usr/libexec/java_home -v 17) ./gradlew assembleDebug
```

## 릴리스(운영) 빌드로 갈 때 (요약)

- 백엔드를 **HTTPS 도메인**으로 배포하고, `VITE_API_BASE_URL` 을 그 주소로 빌드
- 그러면 cleartext 설정 없이 안전하게 동작
- `./gradlew assembleRelease` + 서명 키로 서명 → Play 스토어 업로드용 AAB(`bundleRelease`)
