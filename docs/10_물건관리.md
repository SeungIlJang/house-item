---

## 2. 설계 설명

| 구성 | 내용 |
|------|------|
| 필수/선택 | `home_id` 필수, `room_id`·`storage_location_id`·`category_id`는 선택 |
| 태그 | 다대다(`item_tags` 연결표), `tagIds` 배열로 지정 |
| 삭제 정책 | home 삭제 시 item CASCADE. room/storage/category 삭제 시 해당 FK는 SET NULL(물건은 유지) |
| 일관성 검증 | 방은 그 집 소속, 수납공간은 그 방 소속, 카테고리/태그는 사용자 소유 |
| 날짜 검증 | 유효기간 < 구매일 이면 422 |
| 응답 강화 | `categoryName`, `roomName`, `storageFullPath`, `tags`를 함께 제공(카드 표시용) |
| N+1 방지 | `selectinload`로 연관을 한 번에 로드 |
| 페이지 | 목록은 `PageData`(page/size/totalElements/totalPages) |
