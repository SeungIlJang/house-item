## 1. 이번 단계의 목표

- 집(Home) CRUD, 방(Room) CRUD
- **사용자 소유권 검사**(내 데이터만 조회/수정/삭제)
- 방 정렬 순서(sort_order)
- 통합 테스트

---

## 2. 설계 설명

| 구성 | 내용 |
|------|------|
| 소유권 | Home 은 `user_id`로, Room 은 상위 Home 의 `user_id`로 판단 |
| 없음/권한없음 처리 | 소유자가 아니면 **404**로 통일(자원 존재 여부 노출 방지) |
| 삭제 정책 | Home 삭제 시 Room CASCADE (FK `ondelete=CASCADE` + relationship cascade) |
| 정렬 | Room 목록은 `sort_order, id` 순 |
| 계층 | Router → Service(소유권) → Repository(DB) |
| 라우팅 | 방 목록/생성은 `/homes/{id}/rooms`, 방 단건은 `/rooms/{id}` |
