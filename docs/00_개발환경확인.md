## 3. 항목별 상세

### 3.1 작업 디렉터리
- 경로: `/Users/seungiljang/workspace/house-items`
- 기존 파일: `.python-version`, `README.md`(빈 파일), `main.py`(기본 예제), `pyproject.toml`
- 이 폴더를 **모노레포 루트**로 사용합니다. 여기에 `home-items-api`(백엔드), `home-items-app`(프론트)를 만듭니다.
- 참고: 폴더명은 `house-items`지만 마스터 프롬프트의 프로젝트 영문명은 `home-items`입니다. 내부 프로젝트/패키지 이름은 `home-items`로 통일하고, 폴더는 기존 `house-items`를 그대로 사용합니다.

### 3.2 Git 저장소 — ⚠️ 확인 필요
- `git rev-parse --show-toplevel` 결과: **`/Users/seungiljang/workspace`** (상위 폴더)
- 즉, 현재 Git 저장소가 이 프로젝트만이 아니라 `001_chatbot`, `002_etfbot`, `langgraph-agent` 등 **여러 프로젝트를 함께 담고 있는 상위 폴더**입니다.
- 아직 커밋은 하나도 없습니다(브랜치: `master`).
- **판단이 필요한 부분**: 이 프로젝트(`home-items`)를 위한 **독립 Git 저장소**를 새로 만들지, 상위 저장소를 그대로 쓸지 결정해야 합니다.
  - 권장: `house-items` 폴더에서 `git init`으로 **독립 저장소**를 새로 만드는 것(프로젝트별 이력 관리·배포·협업이 깔끔함). → 2단계에서 진행 예정.

### 3.3 Python — ✅
- 시스템 Python: 3.14.4
- 프로젝트 핀(`.python-version`): 3.13 / `pyproject.toml`의 `requires-python = ">=3.13"`
- FastAPI·SQLAlchemy 2.x 모두 3.13에서 안정 동작. 문제 없음.

### 3.4 uv — ✅
- 0.9.28. 파이썬 가상환경·의존성 관리에 사용합니다. (백엔드 표준 도구)

### 3.5 Node.js / npm — ⚠️ 권장 사항
- Node v18.20.6, npm 10.8.2
- Node 18은 이미 **공식 지원 종료(EOL)** 단계이고, 최신 Vite(7.x)·일부 Ionic 툴체인은 **Node 20 이상**을 요구할 수 있습니다.
- **권장**: Node 20 LTS(또는 22 LTS)로 업그레이드.
  - `nvm` 사용 시:
    ```bash
    nvm install 20
    nvm use 20
    node --version   # v20.x 확인
    ```
- 지금 당장 막히는 것은 아니며, 14단계(프론트 초기화) 전까지만 올리면 됩니다.

### 3.6 Docker / Docker Compose — ❌ 미설치 (의도됨)
- 요구사항: "Docker와 Docker Compose는 **실 배포할 때만** 사용".
- 따라서 개발 중에는 설치하지 않고, **로컬에 이미 떠 있는 PostgreSQL**을 그대로 사용합니다.
- 배포 시점(21단계)에 설치 안내 예정. 설치가 필요해지면:
  - Docker Desktop for Mac 다운로드 후 설치 → `docker --version`으로 확인.

### 3.7 PostgreSQL — ✅ (중요)
- 로컬에 **PostgreSQL 14.15 (Homebrew)** 가 설치되어 **포트 5432에서 실행 중**입니다.
- 덕분에 Docker 없이도 바로 개발용 DB를 쓸 수 있습니다.
- 3단계에서 이 DB에 `home_items` 데이터베이스와 개발용 계정을 만들 예정입니다.
  - 예시(3단계에서 실제 실행):
    ```bash
    createdb home_items
    ```

### 3.8 포트 충돌 — ✅
- 5173(프론트), 8000(백엔드): 비어 있음 → 그대로 사용.
- 5432(DB): 로컬 PostgreSQL이 사용 중이며, 이는 우리가 쓸 DB이므로 정상입니다.

## 5. 다음 단계에서 만들 프로젝트 구조 (미리보기)

```text
house-items/                 # 모노레포 루트 (현재 폴더)
├── home-items-api/          # 백엔드 (FastAPI)
│   └── app/
│       ├── api/             # 라우터 (HTTP 요청/응답)
│       ├── core/            # 설정·보안·예외
│       ├── database/        # DB 세션·Base
│       ├── models/          # SQLAlchemy 모델
│       ├── repositories/    # DB 접근 계층
│       ├── schemas/         # Pydantic 요청/응답
│       ├── services/        # 업무 로직
│       └── main.py
├── home-items-app/          # 프론트 (Ionic Vue)
│   └── src/
│       ├── api/  components/  composables/
│       ├── layouts/  pages/  router/  stores/
│       ├── types/  utils/
│       ├── App.vue
│       └── main.ts
├── docs/                    # 단계별 개발 기록 (이 문서 포함)
├── docker-compose.yml       # (배포 시점에 작성)
├── .env.example
├── .gitignore
└── README.md
```

계층 분리 원칙: **Router → Service → Repository → Database** (Router에서 DB 직접 호출 금지)
