## 2. 테스트 구성 요약

| 파일 | 검증 대상 |
|------|-----------|
| test_health.py | 헬스체크, DB 연결 |
| test_auth.py | 회원가입/로그인/JWT/인증 실패 |
| test_home_room.py | 집·방 CRUD, 소유권 |
| test_storage_location.py | 계층/경로/순환방지/CASCADE/소유권 |
| test_category_tag.py | CRUD, 중복, 사용자 분리, 소유권 |
| test_item.py | CRUD, 링크 검증, 페이지, 날짜검증, 소유권 |
| test_item_image.py | 업로드/삭제/검증/썸네일/소유권 |
| test_item_search.py | 키워드/필터/정렬/페이지 |

- 테스트는 **SQLite 인메모리**(격리) + `PRAGMA foreign_keys=ON`으로 실제와 동일한 CASCADE 동작.
- 소유권/잘못된 입력/경계 케이스를 각 도메인에서 검증.

### 실서버 스모크 테스트 (실제 PostgreSQL, 포트 8000)
```text
/health                        → 200 {"status":"ok","env":"development"}
POST /auth/signup              → 201 사용자 생성
POST /auth/login               → accessToken 발급
POST /homes                    → 집 생성 (id=1)
POST /items                    → 물건 생성 (id=1)
GET  /items/search?keyword=충전 → total 1, "휴대전화 충전기"
```
- 전체 흐름 정상 동작 확인 ✅
- 스모크 데이터는 테스트 후 정리(`DELETE FROM users WHERE email LIKE 'smoke%'` → CASCADE)

> 참고: 셸에서 한글 키워드를 URL 에 직접 넣으면 인코딩 문제가 생길 수 있어
> `curl --data-urlencode` 를 사용해야 정상 동작(서버 자체는 정상).
