## 1. 이번 단계의 목표

- Category CRUD, Tag CRUD
- 이름 중복 검사(사용자 범위 내)
- 사용자별 데이터 분리

---

## 2. 설계 설명

| 구성 | 내용 |
|------|------|
| 소유권 | 각각 `user_id` 기준. 남의 것 접근 시 404 |
| 중복 방지 | `(user_id, name)` UniqueConstraint + 서비스에서 사전 검사(409) |
| 사용자 분리 | 같은 이름도 사용자마다 각자 보유 가능 |
| 구조 | Category/Tag 거의 동일하나 과도한 추상화 없이 각각 명시적으로 구현 |
