# 배포 가이드 (처음 배포하는 분을 위한)

집안의모든것을 **실제 서비스**로 올리는 방법입니다. 사용자가 많지 않은 개인 서비스 기준으로,
**간단하고 저렴한 경로**를 우선 안내합니다.

---

## 0. 전체 그림

배포할 것은 3가지입니다.

```text
[안드로이드 앱] ──HTTPS──▶ [백엔드 API 서버] ──▶ [PostgreSQL DB]
   (Play스토어)              (클라우드에 배포)        (관리형 DB)
                                   │
                                   └──▶ [이미지 저장소] (디스크 또는 오브젝트 스토리지)
```

- **웹 화면**은 앱 안에 이미 포함(번들)되어 있어서, 앱만 낼 거면 **웹을 따로 호스팅할 필요 없습니다.**
  (브라우저에서도 쓰게 하려면 웹도 호스팅 — 아래 5번 참고)
- 핵심은 **백엔드 API를 인터넷(HTTPS)에 올리는 것**입니다.

### 대략의 비용
| 항목 | 비용 |
|------|------|
| Google Play 개발자 등록 | **1회 $25** (필수) |
| 백엔드 서버 | 무료~월 $5~7 (선택한 방식에 따라) |
| PostgreSQL | 무료 티어 가능 (Neon/Supabase 등) |
| 이미지 저장 | 무료 티어 가능 (Cloudflare R2 10GB 무료) 또는 서버 디스크 |
| 도메인(선택) | 연 1~2만원 (없어도 됨 — 제공 도메인 사용 가능) |

## 1. 배포 전에 코드에서 바꿔야 할 것 (중요)

지금은 "개발 PC(LAN)" 기준이라 배포용으로 몇 가지를 바꿔야 합니다. (대부분 제가 도와드릴 수 있음)

1. **이미지 저장 방식** — 지금은 서버 로컬 디스크(`uploads/`)에 저장.
   - 클라우드(Render 등) 무료 티어는 재배포 시 디스크가 초기화되어 **사진이 사라질 수 있음**.
   - 해결: (A) **서버 디스크가 유지되는 방식**(VPS/영구 디스크) 사용, 또는
     (B) **오브젝트 스토리지**(Cloudflare R2/AWS S3)로 변경 — 코드에 `S3FileStorage` 자리 있음(구현 필요).
2. **JWT 비밀키** — 운영용 강력한 무작위 값으로 교체(환경변수).
3. **APP_ENV=production** — 상세 오류 숨김.
4. **CORS_ORIGINS** — 앱/웹의 실제 출처만 허용.
5. **앱을 운영 API 주소로 다시 빌드** — `VITE_API_BASE_URL=https://<운영도메인>/api/v1`, cleartext 끄기(https라 불필요).

## 2. 백엔드 배포 — 두 가지 추천 경로

### 경로 A. 관리형 플랫폼 (클릭 위주, 가장 쉬움) — **초보 추천**

**Render** 예시 (Railway/Fly.io 도 비슷):

1. https://render.com 가입 (GitHub 로그인)
2. **New → Web Service** → GitHub 저장소(`house-item`) 연결
3. 설정:
   - Root Directory: `home-items-api`
   - Environment: **Docker** (우리 `Dockerfile` 사용)
4. **환경변수(Environment)** 추가:
   ```text
   APP_ENV=production
   JWT_SECRET_KEY=<무작위 긴 값>
   DATABASE_URL=<아래 3번 DB 주소>
   CORS_ORIGINS=https://localhost,capacitor://localhost
   UPLOAD_DIR=/app/uploads
   ```
5. **이미지 영구 보관**: Render **Disk** 추가(마운트 경로 `/app/uploads`) — 유료지만 저렴(용량당). 
   또는 R2/S3로 변경(1-①B).
6. 배포되면 `https://<앱이름>.onrender.com` 주소가 생김 → 이게 **운영 API 주소**.

> 무료 웹서비스는 일정 시간 미사용 시 **잠들었다가 첫 요청에 30초쯤 깨어남**(개인용은 감수 가능).

### 경로 B. VPS 1대 + Docker Compose (가장 저렴, 모두 유지됨)

우리 저장소에 이미 `docker-compose.yml`(db+api+app)이 있어서, **서버 1대에 통째로** 올릴 수 있습니다.
이미지도 서버 디스크에 그대로 유지됩니다.

1. **VPS 생성**: DigitalOcean/Vultr/Linode 등에서 월 $4~6짜리 인스턴스(Ubuntu)
2. 도메인 연결(선택) + **Caddy**(자동 HTTPS) 또는 Nginx + Let's Encrypt
3. 서버에서:
   ```bash
   git clone https://github.com/SeungIlJang/house-item.git
   cd house-item
   # .env 에 운영값 설정 (JWT_SECRET_KEY, POSTGRES_PASSWORD 등)
   docker compose up -d --build
   ```
4. Caddy로 `https://<도메인>` → 내부 8080(프론트)·8000(API) 프록시

> 장점: 저렴·이미지 유지·전부 한 곳. 단점: 서버 관리(SSH/보안 업데이트)를 직접.

## 3. PostgreSQL (관리형 무료 티어)

경로 A를 쓰면 DB는 따로 관리형을 붙이는 게 편합니다.

- **Neon** (https://neon.tech) — Postgres 무료 티어, 넉넉함 → 추천
- **Supabase** — Postgres 무료 티어
- Render 자체 Postgres도 있음(무료는 90일 후 유료)

가입 후 만들어진 **연결 문자열**을 `DATABASE_URL` 에 넣습니다. 형식:
```text
postgresql+psycopg://<user>:<password>@<host>/<db>?sslmode=require
```
첫 배포 시 테이블 생성은 우리 Dockerfile 이 시작할 때 `alembic upgrade head` 를 자동 실행합니다.

## 4. 안드로이드 앱 배포 (Google Play)

### 4-1. 운영용 앱 다시 빌드
운영 API 주소로 웹을 빌드하고, https 스킴으로 바꿉니다.
```bash
cd home-items-app
# capacitor.config.ts: androidScheme 'https', cleartext 제거 (운영은 https)
VITE_API_BASE_URL=https://<운영도메인>/api/v1 npm run build
npx cap sync android
```

### 4-2. 서명 키(keystore) 생성 — 최초 1회
```bash
keytool -genkey -v -keystore ~/home-items-release.keystore \
  -alias home-items -keyalg RSA -keysize 2048 -validity 10000
```
> ⚠️ 이 keystore 파일과 비밀번호는 **절대 분실하면 안 됩니다**(앱 업데이트에 필수). 안전하게 백업.

### 4-3. 서명 설정 후 릴리스 번들(AAB) 빌드
`android/app/build.gradle` 에 signingConfig 추가 후:
```bash
cd android && ./gradlew bundleRelease
# 결과: android/app/build/outputs/bundle/release/app-release.aab
```

### 4-4. Play Console 등록
1. https://play.google.com/console 에서 **개발자 계정 등록($25, 1회)**
2. **앱 만들기** → 이름(집안의모든것), 언어, 앱/게임, 무료/유료
3. 스토어 등록정보: 설명, **스크린샷(필수)**, 아이콘(512px), 그래픽 이미지
4. **개인정보처리방침 URL(필수)** — 이메일/사진 수집하므로 필요(초안 제작 도와드릴 수 있음)
5. 콘텐츠 등급 설문, 데이터 보안(수집 항목) 설문 작성
6. **비공개 테스트(Internal testing)** 로 먼저 올려 본인·지인 설치 테스트 → 문제 없으면 **프로덕션** 제출
7. 첫 심사는 보통 며칠 소요

> 팁: 사용자가 적을 거면 **처음엔 Play스토어 대신 APK 직접 배포**(카톡/링크로 설치)도 방법입니다.
> 스토어 등록은 나중에 해도 됩니다.

## 7. 이미지 저장을 Cloudflare R2로 (✅ 구현 완료)

이미지 저장이 **S3 호환(R2 포함)** 으로 전환되었습니다. `.env` 설정만 하면 R2 를 씁니다.

### R2 설정 순서 (클릭 단위)

**1단계. R2 활성화**
- Cloudflare 대시보드 왼쪽 메뉴 → **R2 Object Storage** 클릭
- 처음이면 **결제 정보(신용카드) 등록**을 요구합니다 → 등록
  - ⚠️ 무료 한도(저장 10GB, 요청량 넉넉) 내에서는 **과금되지 않습니다.** 카드는 한도 초과 대비용.
- "Purchase R2 / Enable" 버튼 클릭해 활성화

**2단계. 버킷 만들기**
- **Create bucket** 클릭
- Bucket name: `home-items` (소문자/하이픈)
- Location: **Automatic** (또는 Asia-Pacific)
- **Create bucket**

**3단계. 공개 접근 켜기 (이미지 표시용)**
- 방금 만든 버킷 클릭 → **Settings** 탭
- **Public access** 섹션 → **R2.dev subdomain** → **Allow Access**
  - 경고창에 `allow` 입력 후 확인
- 생성된 **Public bucket URL** 복사 → 예: `https://pub-xxxxxxxxxxxx.r2.dev`
  - 이게 `S3_PUBLIC_BASE_URL` 입니다.

**4단계. API 토큰 발급 (키 2개 + endpoint)**
- R2 개요 화면 오른쪽 위 **{ } API** → **Manage API tokens** (또는 "Manage R2 API Tokens")
- **Create API token** (또는 Create Account API token)
- Token name: `home-items-app`
- Permissions: **Object Read & Write**
- (선택) "Specify bucket(s)" → `home-items` 만 선택하면 더 안전
- **Create API token**
- 표시되는 값 복사(한 번만 보임):
  - **Access Key ID** → `S3_ACCESS_KEY_ID`
  - **Secret Access Key** → `S3_SECRET_ACCESS_KEY`
  - **Endpoint** (`https://<accountid>.r2.cloudflarestorage.com`) → `S3_ENDPOINT_URL`
    - (계정 ID는 R2 개요 화면 오른쪽에도 표시됨)

**5단계. 서버 `.env` 에 입력**
```text
STORAGE_BACKEND=s3
S3_ENDPOINT_URL=https://<accountid>.r2.cloudflarestorage.com
S3_ACCESS_KEY_ID=<Access Key ID>
S3_SECRET_ACCESS_KEY=<Secret Access Key>
S3_BUCKET=home-items
S3_REGION=auto
S3_PUBLIC_BASE_URL=https://pub-xxxxxxxxxxxx.r2.dev
```

**6단계. 서버 재시작 → 업로드 테스트**
- 재시작 후 앱/웹에서 사진을 올리면 R2 에 저장되고, 이미지 URL 은 `S3_PUBLIC_BASE_URL/...` 형태가 됩니다.

- `STORAGE_BACKEND=local` 이면 기존처럼 서버 디스크에 저장(개발 기본값).
- 프론트는 절대 URL(https)·상대 경로 모두 자동 처리하므로 코드 변경 불필요.
- R2 무료 티어: 10GB 저장 + 월 트래픽 넉넉(개인용 충분).

> ⚠️ CORS: 앱/웹이 R2 이미지 URL을 직접 불러오므로, 필요 시 R2 버킷의 CORS 정책에
> 앱/웹 출처를 허용해 두면 안전합니다(공개 버킷 이미지 표시는 대개 문제 없음).
