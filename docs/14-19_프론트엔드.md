## 14단계: Ionic Vue 프론트 초기화

### 목표
Vue 3 + TypeScript + Ionic Vue + Vue Router + Pinia + Axios 기반의 앱 뼈대와 API 클라이언트, 기본 레이아웃 구성.

### 설계 설명
| 구성 | 선택 / 이유 |
|------|-------------|
| 빌드 | Vite 5 (Node 18 호환 위해 최신 대신 안정 버전) |
| UI | Ionic Vue 8 (모바일 우선 컴포넌트) |
| 상태 | Pinia — 전역 인증 상태만 스토어로 |
| 통신 | Axios 인스턴스 + 인터셉터(토큰 자동 첨부, 401 처리) |
| 타입 | 백엔드 응답을 `types/api.ts`에 camelCase 로 정의 |
| API 계층 | `api/index.ts`에서 도메인별 함수로 캡슐화(페이지에 API 로직 분산 방지) |

### 주요 파일
```text
package.json, vite.config.ts, tsconfig.json, index.html, eslint.config.js
src/main.ts, src/App.vue, src/theme/variables.css, src/env.d.ts
src/types/api.ts                 # 공통/도메인 타입
src/api/client.ts                # axios + 토큰/401 인터셉터
src/api/index.ts                 # authApi/homeApi/roomApi/storageApi/categoryApi/tagApi/itemApi
src/stores/auth.ts               # 인증 스토어
src/router/index.ts              # 라우터 + 인증 가드
src/composables/useToast.ts      # 토스트 알림
```

## 15단계: 로그인과 인증 화면

### 목표
회원가입/로그인, JWT 저장, Axios 인증 헤더, 라우터 가드, 로그아웃.

### 토큰 저장 방식 (설명)
- **localStorage** 사용을 선택.
- 장점: 구현 단순, 새로고침/앱 재시작에도 유지, 웹·Capacitor 모두 동작.
- 단점: XSS 에 취약(자바스크립트로 접근 가능). httpOnly 쿠키는 XSS 에 강하지만 CSRF 대비와 서버 세팅이 필요.
- **초기 웹 MVP 판단**: 구현 단순성과 SPA/모바일 호환을 우선해 localStorage 채택.
  운영 강화 시 httpOnly 쿠키 + Refresh Token 으로 이전 가능.

### 구현
- `LoginPage.vue`, `SignupPage.vue`
- `stores/auth.ts`: login/signup/fetchMe/logout, 토큰은 `api/client.ts`의 localStorage 헬퍼로 관리
- `router/index.ts` 가드: 비공개 라우트 + 미인증 → `/login`, 공개 라우트 + 인증 → `/tabs/home`
- 401 응답 시 인터셉터가 토큰 제거 후 `/login` 이동

## 16단계: 집·방·수납공간 화면

- 하단 탭 "보관 장소" → `HomesPage.vue` (집 목록/등록/수정/삭제)
- `RoomsPage.vue` (집 하위 방 관리)
- `StoragePage.vue` (방 하위 수납공간 트리 — 들여쓰기 표시, 하위 추가, 삭제)
- 삭제는 `alertController` 확인 다이얼로그. 슬라이딩 아이템으로 수정/삭제 노출.
- CASCADE 경고 문구 안내(집 삭제 시 하위 함께 삭제).

## 17단계: 물건 등록 화면

- `ItemFormPage.vue` (등록/수정 겸용)
- 집 선택 → 방 목록 로드 → 수납위치(트리 평탄화) 로드 (의존 선택)
- 카테고리 단일 선택, 태그 다중 선택
- 이름/수량/설명/메모/구매일/유효기간 입력
- 사진: 파일 선택 → 저장 시 업로드(수정 화면은 기존 이미지 표시/삭제)
- 입력값 검증(이름·집 필수), 저장 후 등록은 상세로, 수정은 뒤로가기

## 18단계: 물건 검색과 상세 화면

- `SearchPage.vue`: 검색창(디바운스) + 카테고리 칩 필터, 결과 카드 목록, 빈/로딩 상태
- 대시보드 검색창에서 keyword 쿼리로 이동
- `ItemCard.vue`: 대표 사진/이름/카테고리/수량/보관 위치 경로 표시
- `ItemDetailPage.vue`: 이미지 갤러리, 보관 위치 경로, 태그, 설명/메모/날짜, 수정·삭제

## 실행과 검증 결과

```bash
cd home-items-app
npm install
npm run type-check   # 통과 (에러 0)
npm run lint         # 통과 (에러 0)
npm run build        # 성공 (dist 생성)
npm run dev          # http://localhost:5173
```

- `type-check` : 통과 (초기 `node:url` 오류 → `@types/node` 추가로 해결)
- `lint` : 통과 (Ionic `slot` 오탐 규칙 off, .d.ts 제외)
- `build` : 성공 (Ionic 번들로 청크 큼 경고는 MVP 무방)
- **런타임 스모크**: 백엔드(8000)+프론트(5173) 동시 기동 → health OK, 프론트 index.html·main.ts 정상 서빙 확인
